BBD 봉봉단 — 홈페이지 웹 디자인 브리프
================================================================

[1] 프로젝트 개요
----------------------------------------------------------------

봉봉단(BBD)은 "의협을 실천하는 봉사문파"다.
단체 봉사 전문 비영리단체로, 유기견 보호소 봉사를 시작으로 환경·아동·복지 봉사까지 순차 확장하는 조직이다.
2026년 2월 경기도에서 출범, 아임웹 기반 홈페이지를 2026년 3월 공개 예정이다.

핵심 정체성:
- 단체명: 봉봉단(BBD) — 봉사하는 협객들의 모임
- 성격: 의협을 실천하는 봉사문파 · 단체 봉사 전문 비영리단체
- 설립일: 2026년 2월
- 설립자/단장: 박진범
- 도메인: bbd.or.kr
- 플랫폼: 아임웹 기반, 무료 오픈소스 활용
- 기술 스택: 아임웹(호스팅·페이지) + 구글 시트(데이터) + 구글 앱스크립트(자동화)

현재 3가지 문제점 → 홈페이지로 해결:
- 신청 채널 부족 → 구글 설문지 연동으로 신청 자동화
- 기록 관리 어려움 → 구글 시트 자동 기록 + 봉사기록 1초 조회 + PDF 자동 발급
- 홍보 기회 제한 → SEO, 후원 채널, 조직 신뢰도 확보

봉사 분야 로드맵:
- 유기견 봉사: 보호소 산책·환경 정리·홍보 촬영·입양 연결 → 현재 진행 중
- 환경 봉사: 하천 정화·산림 보호·탄소 줄이기 → 준비 중
- 아동 봉사: 지역아동센터·학습 멘토링·놀이 봉사 → 예정
- 복지 봉사: 독거노인·장애인·지역사회 복지 활동 → 예정


[2] 브랜드 정체성
----------------------------------------------------------------

4가지 핵심 가치:
- 봉사의 기쁨: 동물을 돕는 것이 곧 사회 임파워먼트
- 커뮤니티: 혼자가 아니라 함께 성장하는 집단
- 투명성: 모든 활동을 공개하고 신뢰를 쌓는다
- 성장: 함께 배우고 배운 것을 사회에 환원한다

브랜드 톤:
- 무협 세계관을 차용하되 진지하게 사용 (코미디/패러디 아님)
- "협객", "문파", "출정", "무공" 등 무협 용어를 봉사 맥락에 자연스럽게 녹임
- 직급 체계: 봉사 횟수에 따라 단(段) 승급

용어 매핑:
- 단체 → 문파
- 회원/봉사자 → 협객
- 봉사 활동 → 출정
- 봉사 기록 → 무공 기록
- 직급 체계 → 단(段) 체계
- 내부 페이지 → 문파 내부
- 공지사항 → 문파 소식


[3] 디자인 콘셉트
----------------------------------------------------------------

방향: 토스(Toss)의 미니멀 레이아웃 위에 무협(武俠) 세계관 비주얼을 얹는 구조.
깔끔함과 독특함을 동시에 잡되, 무협 요소가 기믹이 아닌 진지한 브랜드 정체성으로 작동해야 한다.

토스에서 가져올 것:
- 넉넉한 여백 (섹션 간 120px+)
- 16px 라운드 카드, 부드러운 그림자 계층 구조
- cubic-bezier(0.16, 1, 0.3, 1) 이징 커브
- 스크롤 트리거 reveal 애니메이션 (IntersectionObserver)
- 숫자 카운터 애니메이션 (easeOutQuart, 2초)
- 최소한의 border, 그림자로 계층 표현

무협에서 가져올 것:
- 먹물 그라데이션 히어로 배경 (네이비 계열)
- 골드 포인트 — 태그 라벨, 디바이더, 장식 요소
- 먹물 번짐(ink splash) 효과 — blur 80~100px 원형 배경 장식
- ⚔ 검 아이콘 — hero-tag 앞뒤 장식
- 무협 용어 자연스러운 혼용

절대 하지 말 것:
- 세로 컬러 바(vertical color bar) 금지
- 카드 스트라이프 패턴 금지
- 제목 밑 무의미한 가로 선 금지
- 스톡 이미지 직접 삽입 금지 (placeholder + 메모만)
- 무협 요소를 코미디/패러디로 쓰지 말 것

[7] 사용자 여정 & 페르소나
----------------------------------------------------------------

페르소나 4명:
- 김서연 (22세, 대학생): 목표=봉사 신청, 고민="혼자 가도 되나?"
- 이준호 (24세, 직장인, 기존단원): 목표=기록 확인+PDF, 고민="복잡하면 귀찮"
- 박미영 (45세, 주부, 후원자): 목표=후원 방법 파악, 고민="돈이 잘 쓰이나?"
- 정태윤 (26세, 운영진): 목표=콘텐츠 업데이트, 고민="아임웹 어려울까?"

사용자 유형별 비율:
- 60% 신규 방문자: 3초 안에 '뭐하는 곳' 파악, 봉사 신청 클릭 3회 이내
- 25% 기존 단원: 봉사 기록 조회 + PDF, 빠른 접근성
- 10% 후원자: 후원 방법 명확 안내, 투명한 사용 내역
- 5% 운영진: 아임웹 + 구글 시트 관리, 콘텐츠 업데이트

사용자 여정 ① (신규 방문자 → 봉사 신청):
봉봉단 발견(SNS/지인) → 홈페이지 첫 방문(3초 파악) → 소개·봉사 둘러보기 → 봉사 신청 클릭(구글 설문지) → 운영진 카카오 안내 → 첫 봉사

사용자 여정 ② (기존 단원 → 기록 확인):
내 봉사 기록 접속 → 이름+생년월일 입력 → 횟수·직급 확인 → PDF 발급 클릭 → 다운로드 완료 (약 5~8초)

사용자 여정 ③ (후원자 → 후원):
후원 페이지에서 필요성 확인 → 현금 후원(계좌 이체) / 물품 기부(카카오 연락) / 재능 기부(이메일 문의)

사용자 여정 ④ (운영진 → 관리):
봉사 기록 등록(구글 시트) → 공지사항 작성(아임웹 블로그) → 갤러리 업로드 → 회원 승급 처리(비밀번호 카카오 전달)

감정 여정 (신규 방문자):
발견(궁금) → 첫방문(흥미) → 탐색(탐색) → 신청(약간 긴장) → 첫봉사(뿌듯, Peak!)


[8] 특수 기능 (4가지 자동화)
----------------------------------------------------------------

① 카운터 자동화:
- 구글 시트 ↔ 홈 카운터
- 누적 봉사자 / 시간 / 동물 수
- 운영진이 수동 입력만 하면 카운터 자동 업데이트

② 신청 자동기록:
- 구글 설문지 → 자동 시트 기록
- 신청자 정보 자동 수집
- 운영진 확인만 하면 됨

③ PDF 자동발급:
- 조회 → 자동 증명서 생성
- 새 탭에서 열림, 다운로드 가능
- 봉봉단 로고 포함
- 대학교 봉사활동 인증용

④ 카카오 알림:
- 신청/확인 시 자동 안내
- 실시간 카카오톡 알림

신청 프로세스:
홈에서 신청 클릭(버튼 3곳) → 구글 설문지(이름, 생년월일, 일정, 카카오ID) → 자동 시트 기록 → 운영진 카카오톡 안내

PDF 발급 프로세스:
'내 봉사 기록' 접속 → 이름+생년월일 입력 → 구글 시트 자동 검색 → 결과 자동 채움 → PDF 변환 → 새 탭 열림
